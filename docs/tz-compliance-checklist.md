# Перевірка відповідності ТЗ SPA «Коментарі»

Останнє оновлення: 2026-03-18.

## Рішення по БД (оновлено)

- ✅ Прийнято рішення переходити з MS SQL на SQLite для цього проєкту.
- ✅ Для локальної розробки окремий сервер БД **не потрібен**: SQLite працює як файлова БД (`.db`) у застосунку.
- ⚠️ Для production потрібно окремо зафіксувати обмеження SQLite (конкурентний запис, backup-стратегія, масштабування) і критерії, коли потрібен перехід на серверну БД.

## Що внесено (ця ітерація)

- ✅ Видалено неактуальні пункти про MySQL/Docker healthcheck як основний сценарій.
- ✅ Чекліст перефокусовано під міграцію на SQLite.
- ✅ Додано правило документування: при редагуванні або створенні нових класів/методів обов'язково додавати коментарі (для C# публічних елементів — XML-коментарі `///`).

## Що ще потрібно зробити у проєкті (актуальний backlog для SQLite)

1. 🔜 **Конфігурація провайдера EF Core**
   - Додати/увімкнути `Microsoft.EntityFrameworkCore.Sqlite`.
   - Перемкнути `Persistence:Provider` на `Sqlite`.
   - Додати `ConnectionStrings:CommentsDb` у форматі `Data Source=...`.

2. 🔜 **Оновити `Program.cs` (гілка persistence-провайдера)**
   - Додати гілку `Sqlite` з `options.UseSqlite(...)`.
   - Позначити статус `SqlServer`-гілки (залишаємо як fallback чи видаляємо).

3. 🔜 **Міграції БД**
   - Створити новий набір EF Core міграцій для SQLite.
   - Перевірити типи полів/індекси, сумісність datetime/string-обмежень з SQLite.
   - Прогнати `database update` на чистій БД.

4. 🔜 **Інфраструктурний код під старт застосунку**
   - Видалити або спростити MySQL-specific preflight (`host/port` очікування), бо для SQLite він не потрібен.
   - Оновити логування startup-помилок під файлову БД.

5. 🔜 **Docker / середовище виконання**
   - Прибрати залежність API від контейнера серверної БД (якщо лишається лише SQLite).
   - Додати volume для збереження `.db` файлу між перезапусками контейнера.

6. 🔜 **Тести і перевірки**
   - Оновити інтеграційні тести на SQLite-сценарій.
   - Додати smoke-check запуску API з новими налаштуваннями.

7. 🔜 **Документація**
   - Оновити `README.md`: нові інструкції запуску без встановлення MS SQL.
   - Додати розділ про обмеження SQLite і сценарії, коли варто повернутись до серверної БД.
   - Зафіксувати правило: будь-які нові/змінені класи і методи мають супроводжуватись коментарями.
