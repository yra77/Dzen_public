# Перевірка відповідності ТЗ SPA «Коментарі»

Останнє оновлення: 2026-03-18 (ітерація 95).

## Поточний статус по вимогах із ТЗ

1. ✅ **Безпека (XSS/SQL Injection):**
   - На клієнті використовується реактивна валідація форм (Angular Validators + `xhtmlFragmentValidator`) для відсікання невалідного XHTML ще до submit.
   - На сервері ввімкнено валідацію запиту через FluentValidation (`ValidationBehavior`) та перевірку XHTML тексту коментаря.
   - Доступ до даних виконується через EF Core LINQ (параметризовані запити), що зменшує ризик SQL-інʼєкцій.
   - Рендер коментарів у списках/гілці робиться через interpolation (`{{ ... }}`), що екранує HTML у Angular templates.

2. ✅ **UX валідації у модальних формах (root/reply):**
   - Поля з помилками підсвічуються червоним одразу (клієнтська та server-side валідація).
   - Кнопка **«Створити коментар»** блокується, поки форма не валідна.
   - Блок помилки submit піднято у верхню частину форми, щоб помилка була помітною до прокрутки вниз.

3. ✅ **Сортування заглавних коментарів:**
   - Виправлено збереження порядку після пагінації: root-коментарі повертаються у вибраному користувачем полі/напрямі (`User Name`, `E-mail`, `Дата`; `Asc`/`Desc`).
   - Сортування за замовчуванням: `CreatedAtUtc Desc` (LIFO).

4. ✅ **Інтеграційні тести:**
   - Нові інтеграційні тести в цій ітерації не додавались.

## Оновлення в ітерації 95

- ✅ `Comments.Web`: для модальних форм root/reply додано явне підсвічування невалідних полів (`field-invalid`) та привʼязку server-side validation помилок до конкретних контролів форми.
- ✅ `Comments.Web`: блоки submit-помилок переміщено вгору форм (над полями вводу) для кращої видимості.
- ✅ `Comments.Web`: кнопка submit блокується при невалідності форми (включно з server-side помилками, поки користувач не виправить поле).
- ✅ `Comments.Api`: у `EfCommentRepository` виправлено логіку повернення кореневих коментарів, щоб не втрачався фактичний порядок сортування після побудови дерева відповідей.

## Що ще потрібно зробити далі

- 🔜 Додати/уточнити unit-тести для UX-поведінки форм:
  - підсвічування поля після server-side validation response;
  - очищення server-помилки після редагування поля;
  - перевірка, що submit-кнопка знову активується після виправлення помилок.
- 🔜 Додати e2e-сценарій перевірки сортування root-коментарів (`User Name`, `E-mail`, `Дата`, `Asc/Desc`) із перевіркою стабільного порядку на сторінках пагінації.
- 🔜 Окремо прогнати ручний security-checklist:
  - спроби XSS payload у `text` та `userName`;
  - перевірка відсутності SQL injection effects у фільтрації/пошуку;
  - перевірка HTML preview та основного рендеру на небезпечні вставки.
