# Перевірка відповідності ТЗ SPA «Коментарі»

Останнє оновлення: 2026-03-18.

## Поточний статус по вимогах із ТЗ

1. ✅ **Безпека (XSS/SQL Injection):**
   - На клієнті застосовується реактивна валідація форм (`Validators` + `xhtmlFragmentValidator`) до відправки.
   - На сервері використовується FluentValidation (`ValidationBehavior`) і санітизація/перевірка тексту.
   - Доступ до БД виконується через EF Core LINQ, без ручної конкатенації SQL.
   - У шаблонах Angular рендер тексту коментарів виконується через interpolation (`{{ ... }}`).

2. ✅ **UX валідації у формах створення коментарів:**
   - Поля з помилками підсвічуються (`field-invalid`) для client-side і server-side валідації.
   - Кнопки submit блокуються для невалідної форми або під час submit.
   - Помилки submit показуються у верхній частині модального вікна.

3. ✅ **Сортування:**
   - Сортування застосовується тільки до **головних (root) коментарів** у списку.
   - Відповіді в гілках більше не перевпорядковуються за `UserName`/`Email`; вони відображаються за датою створення (від старих до нових).
   - Сортування root-коментарів за замовчуванням: `CreatedAtUtc Desc`.

4. ✅ **Інтеграційні тести:**
   - Нові інтеграційні тести у проєкт не додаються (за поточним узгодженим обмеженням).

## Що змінено в цій ітерації

- ✅ Оновлено бізнес-логіку мапінгу дерева коментарів: сортування за полями списку (`UserName`/`Email`) більше не застосовується до reply-вузлів.
- ✅ Для thread-відображення залишено стабільний порядок відповідей за `CreatedAtUtc Asc`.
- ✅ Документацію відповідності ТЗ актуалізовано: прибрано неактуальні згадки про попередні ітерації та зафіксовано поточні правила сортування.

## Що ще потрібно зробити у проєкті

- 🔜 Додати/оновити unit-тести для `CommentService`, які перевіряють правило: сортування стосується тільки root-коментарів, а replies мають фіксований порядок за датою.
- 🔜 Перевірити UI-поведінку списку root-коментарів при перемиканні `sortBy`/`sortDirection` на кількох сторінках пагінації.
- 🔜 Провести ручний regression-check створення відповідей у гілці (щоб порядок відповідей лишався стабільним після створення нової reply).
- 🔜 Зафіксувати в README коротке правило: «параметри сортування впливають лише на кореневий список».
